# ข้อมูลจำเพาะทางเทคนิค

ข้อมูลจำเพาะทางเทคนิคของแต่ละส่วนประกอบที่ใช้ในโครงการ Autobot

---

## ไมโครคอนโทรลเลอร์ — ESP32

ESP32 ทำหน้าที่เป็นหน่วยควบคุมหลักของหุ่นยนต์

| ข้อมูลจำเพาะ | ค่า |
| :--- | :--- |
| **รุ่นที่รองรับ** | ESP32 DevKitC, ESP32 NodeMCU |
| **โปรเซสเซอร์** | Dual-core Xtensa LX6 ความเร็วสูงสุด 240 MHz |
| **การเชื่อมต่อ** | Wi-Fi 802.11 b/g/n, Bluetooth 4.2 |
| **ขา GPIO** | สูงสุด 34 ขาที่โปรแกรมได้ |
| **การสื่อสาร** | UART, SPI, I2C, PWM |
| **แรงดันไฟทำงาน** | 3.3V (อินพุต 5V ผ่าน USB หรือ VIN) |
| **หน่วยความจำ Flash** | 4 MB |

---

## มอเตอร์

ระบบรองรับ **มอเตอร์ DC ทุกประเภท** ที่สามารถควบคุมผ่านไดรเวอร์แบบ H-Bridge ได้ แม้ว่าโครงการจะแนะนำ TT Motor สำหรับผู้เริ่มต้น แต่ผู้ใช้งานไม่ได้ถูกจำกัดอยู่แค่นั้น

| ข้อมูลจำเพาะ | ค่า / ข้อกำหนด |
| :--- | :--- |
| **ประเภทที่รองรับ** | มอเตอร์ DC เกียร์, มอเตอร์ DC แปลงถ่าน (เช่น TT Motor, N20, JGA25) |
| **แรงดันไฟทำงาน** | ต้องอยู่ในขีดจำกัดของไดรเวอร์มอเตอร์ (เช่น 2.5V – 12V สำหรับ L9110S) |
| **การกินกระแสไฟ** | ต้องไม่เกินกระแสสูงสุดของไดรเวอร์มอเตอร์ (เช่น < 800mA สำหรับ L9110S) |
| **วิธีการควบคุม** | PWM สำหรับความเร็ว, สัญญาณ Digital สำหรับทิศทาง (ผ่าน H-Bridge) |
| **รองรับ Encoder** | เป็นทางเลือก สามารถรองรับได้หาก MCU มีขา GPIO เพียงพอสำหรับรับสัญญาณ |

---

## ไดรเวอร์มอเตอร์ — L9110S

| ข้อมูลจำเพาะ | แบบ 2 ช่อง | แบบ 4 ช่อง |
| :--- | :--- | :--- |
| **ประเภท** | Dual H-Bridge | Quad H-Bridge |
| **แรงดันไฟทำงาน** | 2.5V – 12V | 2.5V – 12V |
| **กระแสสูงสุดต่อช่อง** | 800 mA | 800 mA |
| **วิธีการควบคุม** | PWM สำหรับความเร็ว, Digital สำหรับทิศทาง | PWM สำหรับความเร็ว, Digital สำหรับทิศทาง |
| **มอเตอร์ที่รองรับ** | 2 ตัว | 4 ตัว |
| **เหมาะสำหรับ** | หุ่นยนต์ 2 ล้อ, 3 ล้อ | หุ่นยนต์ 4 ล้อ |

> **หมายเหตุ:** ผู้ใช้งานสามารถใช้ไดรเวอร์มอเตอร์แบบ H-Bridge อื่นๆ (เช่น L298N, TB6612FNG) ได้ หากต่อสายเข้ากับ ESP32 ถูกต้อง และปรับแต่งแหล่งจ่ายไฟให้เหมาะสม

---

## ล้อ

ผู้ใช้งานสามารถใช้ **ล้อประเภทใดก็ได้** ที่สามารถติดตั้งเข้ากับมอเตอร์และโครงสร้างของหุ่นยนต์ โดยโค้ดควบคุมสามารถปรับแต่งให้รองรับจลนศาสตร์ (Kinematics) ในรูปแบบต่างๆ ได้

| ประเภทล้อ | การใช้งานทั่วไป | คุณสมบัติหลักทางการเคลื่อนที่ |
| :--- | :--- | :--- |
| **ล้อยาง / มาตรฐาน (Differential)** | 2W, 4W | เดินหน้า/ถอยหลัง, หมุนด้วยความเร็วล้อที่ต่างกัน |
| **ล้อออมนิ (Omni)** | 3W | เคลื่อนที่ได้หลายทิศทาง |
| **ล้อแมคคานัม (Mecanum)** | 4W | เคลื่อนที่ได้ทุกทิศทาง (รวมด้านข้าง) |
| **ล้อ Caster / Ball** | ล้อค้ำยัน | หมุนได้อิสระ ช่วยทรงตัวสำหรับหุ่นยนต์ 2 ล้อ |

## บอร์ดควบคุม — PCB ออกแบบเฉพาะ

| ข้อมูลจำเพาะ | ค่า |
| :--- | :--- |
| **MCU ที่รองรับ** | ESP32 DevKitC, ESP32 NodeMCU |
| **Co-Processor** | Raspberry Pi 5 |
| **ส่วนประกอบ** | จ่ายไฟ, ควบคุม, LED แสดงสถานะ |
| **ไฟ LED** | 🔴 ไฟเลี้ยง, 🟢 Wi-Fi, 🟡 กำลังทำงาน, ⚪ สำรอง |
| **อินพุตไดรเวอร์มอเตอร์** | รองรับ L9110S (2-ch หรือ 4-ch) |
| **การยึด** | ตรงกับรูยึด Raspberry Pi 5 |

## แหล่งจ่ายไฟ

| แหล่งจ่ายไฟ | แรงดัน | ต้องใช้ Step-Down | หมายเหตุ |
| :--- | :--- | :--- | :--- |
| **18650 Li-Ion** | 3.7V – 4.2V (ต่อเซลล์) | ✅ ต้องใช้ | กะทัดรัด ชาร์จซ้ำได้ |
| **ถ่าน AA** | 1.5V ต่อก้อน | ✅ ต้องใช้ (ต่ออนุกรม) | หาซื้อได้ง่าย |
| **Power Bank** | 5V | ❌ ไม่ต้อง | ต่อตรงได้เลย |
| **Li-Po** | 3.7V – 7.4V | ✅ ต้องใช้ | กระแสสูง น้ำหนักเบา |

> **สำคัญ:** ระบบต้องการไฟ **5V DC** เสมอ ใช้วงจร Step-Down หากแหล่งจ่ายไฟมีแรงดันเกิน 5V

## Co-Processor — Raspberry Pi 5

| ข้อมูลจำเพาะ | ค่า |
| :--- | :--- |
| **โปรเซสเซอร์** | Broadcom BCM2712, Quad-core Cortex-A76 @ 2.4GHz |
| **RAM** | 4GB / 8GB LPDDR4X |
| **การเชื่อมต่อ** | Wi-Fi 5, Bluetooth 5.0, Gigabit Ethernet |
| **กล้อง** | CSI connector / รองรับ USB camera |
| **ความสามารถ AI** | ตรวจจับวัตถุด้วย YOLOv8, ติดตามเส้น |
| **การสื่อสาร** | UART (GPIO 14/15) กับ ESP32 |
| **พลังงาน** | 5V / 5A ผ่าน USB-C |
