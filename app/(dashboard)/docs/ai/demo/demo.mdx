# Examples

This page provides complete, ready-to-use sketches for common Autobot configurations. Each example demonstrates a different drive type and feature set.

## Installation

1. Download [`ArduinoRobot.h`](https://github.com/CannabiZz9/Autobot_Library/blob/main/ArduinoRobot.h).
2. Place the file in your Arduino project folder.
3. Include it at the top of your sketch:

```cpp
#include "ArduinoRobot.h"
```

## Differential Drive (2-Wheel)

### Differential With Raspberry Pi

> [Source Code](https://github.com/CannabiZz9/Autobot_Library/tree/main/Arduino-client-diff)

```cpp
#include "ArduinoRobot.h"

// ── Pin Configuration ──────────────────────────
int motorPins[] = {18, 19, 22, 23};

// ── Serial Configuration (Raspberry Pi) ────────
HardwareSerial PiSerial(2);

#define PI_RX_PIN 16
#define PI_TX_PIN 17

// ── Objects ────────────────────────────────────
ArduinoClient robot;
PiCommunication piComm(PiSerial, robot);

void setup() {
  // Start USB serial for debugging
  Serial.begin(115200);
  Serial.println("ESP32 Ready.");

  // Start serial communication with the Pi
  PiSerial.begin(115200, SERIAL_8N1, PI_RX_PIN, PI_TX_PIN);

  // Initialize the robot (Differential, 2 motors, no deviation)
  robot.init(DRIVE_DIFFERENTIAL, motorPins, 2, 0);

  // Initialize communication handler
  piComm.init();
}

void loop() {
  piComm.handle();
}
```

## Mecanum Drive (4-Wheel)

### Mecanum With Raspberry Pi

> [Source Code](https://github.com/CannabiZz9/Autobot_Library/tree/main/Arduino-client-mac)

```cpp
#include "ArduinoRobot.h"

// ── Pin Configuration (Mecanum) ────────────────
const uint8_t FL_IN1 = 12; const uint8_t FL_IN2 = 13;  // Front Left
const uint8_t FR_IN1 = 23; const uint8_t FR_IN2 = 22;  // Front Right
const uint8_t RL_IN1 = 32; const uint8_t RL_IN2 = 33;  // Rear Left
const uint8_t RR_IN1 = 21; const uint8_t RR_IN2 = 19;  // Rear Right

// Pin Array: [FL1, FL2, FR1, FR2, RL1, RL2, RR1, RR2]
int motorPins[] = {FL_IN1, FL_IN2, FR_IN1, FR_IN2, RL_IN1, RL_IN2, RR_IN1, RR_IN2};

// ── Serial Configuration (Raspberry Pi) ────────
HardwareSerial PiSerial(2);

#define PI_RX_PIN 16
#define PI_TX_PIN 17

// ── Objects ────────────────────────────────────
ArduinoClient robot;
PiCommunication piComm(PiSerial, robot);

void setup() {
  Serial.begin(115200);
  Serial.println("ESP32 Ready.");

  PiSerial.begin(115200, SERIAL_8N1, PI_RX_PIN, PI_TX_PIN);
  PiSerial.setTimeout(5);  // Reduce timeout to minimize lag

  robot.init(DRIVE_MECANUM, motorPins, 4, 0);

  piComm.init();
  piComm.requestHuman();  // Start Human Tracking immediately
}

void loop() {
  piComm.handle();
}
```

### Timed AI Sequence

> [Source Code](https://github.com/CannabiZz9/Autobot_Library/blob/main/Demo-Timing-AI/Arduino-client-mac.ino)

This example demonstrates how to chain multiple AI modes using timers:

1. **Phase 1** — Follow a human for **10 seconds**.
2. **Phase 2** — Follow a line for **15 seconds**.
3. **Phase 3** — Stop.

```cpp
#include "ArduinoRobot.h"

// ── Pin Configuration (Mecanum) ────────────────
const uint8_t FL_IN1 = 12; const uint8_t FL_IN2 = 13;
const uint8_t FR_IN1 = 23; const uint8_t FR_IN2 = 22;
const uint8_t RL_IN1 = 32; const uint8_t RL_IN2 = 33;
const uint8_t RR_IN1 = 21; const uint8_t RR_IN2 = 19;

int motorPins[] = {FL_IN1, FL_IN2, FR_IN1, FR_IN2, RL_IN1, RL_IN2, RR_IN1, RR_IN2};

// ── Serial Configuration (Raspberry Pi) ────────
HardwareSerial PiSerial(2);

#define PI_RX_PIN 16
#define PI_TX_PIN 17

// ── Objects ────────────────────────────────────
ArduinoClient robot;
PiCommunication piComm(PiSerial, robot);

// ── Timing State ───────────────────────────────
unsigned long startTime;
bool phaseLineStarted = false;
bool phaseStopStarted = false;

void setup() {
  Serial.begin(115200);
  Serial.println("ESP32 Ready. Starting automated sequence...");

  PiSerial.begin(115200, SERIAL_8N1, PI_RX_PIN, PI_TX_PIN);
  PiSerial.setTimeout(5);

  robot.init(DRIVE_MECANUM, motorPins, 4, 0);
  piComm.init();

  // Phase 1: Start Human Tracking
  Serial.println("PHASE 1: Human Tracking (10s)");
  piComm.requestHuman(30, 0.6, 30000, 5000);

  startTime = millis();
}

void loop() {
  piComm.handle();

  unsigned long elapsed = millis() - startTime;

  // After 10s → switch to Line Tracking
  if (!phaseLineStarted && elapsed > 10000) {
    Serial.println("PHASE 2: Line Tracking (15s)");
    piComm.requestLine(30, 0.8, 100, "black");
    phaseLineStarted = true;
  }

  // After 25s (10 + 15) → stop
  if (!phaseStopStarted && elapsed > 25000) {
    Serial.println("PHASE 3: Sequence complete. Stopping.");
    piComm.requestStop();
    phaseStopStarted = true;
  }
}
```

## AI Mode Quick Reference

You can call these functions anywhere in your `loop()` — inside `if`, `else`, `while`, timers, or conditional blocks.

### Line Tracking

```cpp
piComm.requestLine(30, 0.8, 100, "black");
```

The robot will begin following a line. See the [API Reference](/docs/ai/reference) for full parameter details.

### Human Tracking

```cpp
piComm.requestHuman(30, 0.6, 30000, 5000);
```

The robot will begin following a detected human. See the [API Reference](/docs/ai/reference) for full parameter details.
