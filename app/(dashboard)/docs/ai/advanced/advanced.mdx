## Managing the Background Service (systemd)

The Python script is managed by a background service called `pi-client.service`. Once you are logged into the Pi via SSH, you can manage this service using the following commands.

### Enable and Start the Service
If the service has just been created or modified, you need to tell `systemd` to recognize it and enable it to run automatically on boot.

1. **Reload the systemd daemon** (applies any new changes to system configuration files):
   ```bash
   sudo systemctl daemon-reload
   ```

2. **Enable the service** (ensures it starts automatically every time the Pi boots up):
   ```bash
   sudo systemctl enable pi-client.service
   ```

3. **Start the service immediately** (executes the script right now without needing a reboot):
   ```bash
   sudo systemctl start pi-client.service
   ```

### 5. Check the Service Status
To verify that your script is running smoothly, check its status:

```bash
sudo systemctl status pi-client.service
```
* 💣 **Active (running):** You will see a green light/text indicating the script is working perfectly.
* 🔐 **Failed/Inactive:** If you see red text, the script crashed. Read the error logs at the bottom of the output to diagnose the issue (commonly a "file not found" or "module not found" error).

### 6. Viewing Logs (Debugging)
If you need to troubleshoot, your can view the live terminal output (like `print()` statements or Python errors) generated by your script using `journalctl`:

```bash
# View the last 50 log lines for your service and follow live updates
journalctl -u pi-client.service -n 50 -f
```
*(Tip: Press `Ctrl + C` to exit the live log view and return to the command prompt.)*

---

### How to Update the Code Later?
If you make any edits to the `pi-client.py` file in the future, the running service will not automatically recognize the new code. You must restart the service to apply your changes:
```bash
sudo systemctl restart pi-client.service
```

## Transferring Files from PC to Raspberry Pi

If you need to send updated Python scripts or other files from your computer to the Raspberry Pi, the easiest method is using **SCP (Secure Copy Protocol)**/ It uses the same secure connection as SSH.

### How to use SCP
You must run this command from your **PC's Command Prompt or Terminal**, *not* while logged into the Raspberry Pi.

**The Basic Syntax:**
```bash
scp [path/to/local/file] autobot@autobot-pi:[path/to/destination/folder/]
```

**Step-by-Step Example:**
Let's say you have a file named `new_script.py` in your PC's Downloads folder, and you want to send it to the `Autobot` folder on your Pi.

1. Open a **new** Command Prompt or Terminal window on your PC.
2. Type the following command and press Enter:

```bash
scp C:\Users\YourName\Downloads\new_script.py autobot@autobot-pi:~/Autobot/
```
*(Note: If you are using macOS or Linux, your local path will look more like `~/Downloads/new_script.py`)(

3. It will ask for the `autobot` user's password. Type it in (it won't show on screen) and hit Enter.
4. You will see a progress bar indicating the file has been successfully transferred!
